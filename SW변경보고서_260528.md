## 2. 전체 동작 흐름 (검증 기준)

```
[BMS OFF→ON]  System_STATE_INIT (1회)
   HW init(Clock/GPIO/SPI/CAN/CpuTimer0=1ms) → CalSocRegsInit() → 셀 전압·온도 측정
   → NVRAM LastSOC 읽기+유효성 검증(음수/범위초과/첫부팅 → 무효마커 -1.0F)
   → CellP56AhSocInit() : 셀평균전압→OCV→SOC(LUT), NVR vs OCV delta(20%) 판정 → SysSocInitF
   → SysSOCF = SysSocInitF → STANDBY

[STANDBY→READY]  CANCOMEnable=1, INITOK=1
   → 이후 메인루프: INITOK==1 이면 매 사이클 CANCOMEnable=1 강제 (통신 끊겨도 송신 유지) [#3]

[운용 중]  1ms ISR, INITOK==1
   Cal80VSysCurrentHandle() → CalSocHandle() : 전류적산만(50ms 게이트, OCV 재보정 없음) [#4]
   |I|<0.5A deadband → 정지 중 SOC 동결(점프/드리프트 방지)
   ΔAh=I×dt 누적 → SysSOCF = SysSocInitF + 적산분 → Bat80VSOCF 출력 + 주기적 NVRAM 저장

[PWRHOLD]  1ms ISR, 독립 (PWRHoldHandle)
   RUNStatus=1            → PWRHOLD_ON
   RUNStatus=0 & 통신정상  → PWRHOLD_ON (SOC는 전류적산 계속)
   RUNStatus=0 & 통신끊김  → 5분 카운트 → PWRHOLD_OFF
   ※ 통신 끊기면 워치독이 RUNStatus=0 자동 강제 [main.c:960]

[전원 재투입 시나리오]  ← 변경 사유 핵심
   5분 내 OFF→ON  : PWRHOLD 유지 중 → 리부팅 안 함 → OCV 재초기화 없음 → SOC 연속(점프 방지)
   5분 초과 종료   : PWRHOLD_OFF → 셀 전압 안정화 후 종료 → 다음 부팅 시 안정 OCV로 초기화
```

핵심 구조: **부팅 1회 OCV 초기화 → 운용 중 전류적산만 → PWRHOLD가 짧은 전원 단절을 흡수**하여 셀 전압 안정화 전 재초기화로 인한 SOC 점프를 차단한다.

## 3. 변경 상세 (코드 위치)

### 3.1 하드웨어 사양 (`parameter.h`, `main.c`, `DSP28x_Project.c`)

| 항목 | 변경 전 | 변경 후 |
|------|---------|---------|
| 셀 직렬 (`Product_SysCellVauleS`) | 24S | **22S** |
| 팩 상한 (`Bat80VSysVoltMax`) | 1008 | **924** (4.2×22) |
| 팩 하한 (`Bat80VSysVoltMin`) | 672 | **616** (2.8×22) |
| 온도센서 (`Sys80VCellTempCount`) | 24 | **22** |
| ISR (`ISR_FREQUENCY`) | 10 kHz | **1 kHz** |
| 클럭 (`CPUCLK`/`SYSTEM_FREQUENCY`) | 90 MHz | **80 MHz** (`CPU_CLOCK_SPEED` 12.5) |
| 셀전압 복사 (`main.c`) | 12+12 | **11+11 = 22** |
| 12V 시스템 | 존재 | **전면 제거** (파라미터/함수/`Slave3Regs`) |

### 3.2 SOC 초기화 — `CellP56AhSocInit` ([BATAlgorithm.c](SysSoure/BATAlgorithm.c)) [#1]

- OCV→SOC를 **17포인트 Lookup Table + 선형보간**(`OCVtoSOC_P56Ah`)으로 교체. 셀 SOC 10~90%(DOD 80%)를 시스템 SOC 0~100%로 매핑, cliff 영역 ±0.5%.
- **NVR 유효성 판정**: 미초기화(`0xFFFF`)/손상(`0`)/범위 이탈 → 무효마커 `-1.0F` → OCV 강제. 선형영역에서 NVR–OCV `delta>20%` 시에도 OCV 우선.

### 3.3 CAN 통신 상시 유지 ([main.c:231-233](SysSoure/main.c#L231-L233)) [#3]

- 메인 while 루프 진입부에서 `INITOK==1`이면 `CANCOMEnable=1` 강제.
- 워치독 ([main.c:957-963](SysSoure/main.c#L957-L963)): 100ms×10=1초 미수신 시 `RUNStatus=0`, `PackCAN_ERR=1`, `SysCanRxCount=11000` clamp.
- CAN RX ISR ([main.c:1208](SysSoure/main.c#L1208)): VCU(ID 0x450) 수신 시 `SysCanRxCount=0` 리셋.

### 3.4 SOC 운용 — `CalSocHandle` ([BATAlgorithm.c:324-387](SysSoure/BATAlgorithm.c#L324-L387)) [#4]

- **운용 중 OCV 재보정 로직 완전 제거** (기존 5분 정지 후 OCV 재초기화 삭제). 부팅 1회 초기화 이후 SOC는 `SysSocInitF + 전류적산`으로만 갱신, `CalMeth` 항상 1.
- 50ms 게이트(1ms ISR×50). 전류 deadband `C_SocCurrentDeadbandF=0.5A`: `|I|<0.5A` → 적분 차단(정지 SOC 표류 방지). `±C_SocAvailableCapacityAh` clamp.

### 3.5 PWRHOLD 전원 유지 제어 — `PWRHoldHandle` (신규)

- 상수 ([parameter.h](SysInclude/parameter.h)): `C_VcuCommLostCount=10u`(1초), `C_PwrHoldOffDelayCount=300000UL`(1ms×30만=5분, `UL` 필수).
- 호출: [main.c:1157](SysSoure/main.c#L1157) `cpu_timer0_isr`(1ms) 내.
- 로직: `RUNStatus=1`→`PWRHOLD_ON`+카운트 리셋 / `RUNStatus=0`&통신끊김→5분 후 `PWRHOLD_OFF` / `RUNStatus=0`&통신정상→`PWRHOLD_ON`.

### 3.6 버그 수정·정리

| 파일 | 내용 |

## 5. 검증 시 주의·확인 필요 항목 (Open Items)

1. **`PWRHoldHandle`의 `PwrOffDelayCount`가 `static`** — `ProtectRelayVarINIT`/`SysVarINIT`에서 리셋되지 않음. RUN/통신정상 시 자동 0 리셋되어 self-heal 되나, INIT 경유 시 잔존값 영향 없는지 D4/D5에서 확인.
2. **워치독 매직넘버 불일치 가능성** — [main.c:958](SysSoure/main.c#L958)은 `>=10` 하드코딩, `PWRHoldHandle`은 `C_VcuCommLostCount` 사용. 동일 의미이나 상수 단일화 권장.
3. **`C_SocOCVLinearMaxF` 중복 정의** — [BATAlgorithm.h:81-82](SysInclude/BATAlgorithm.h#L81-L82) 동일 값 2회. 1줄 삭제 권장.
4. **NVR 클램프 로직 중복** — `main.c`의 NVR 유효성/클램프가 `CellP56AhSocInit` 내부와 중복. 단일화 검토.
5. **미사용 상수 가능성** — OCV 재보정 제거로 `C_SocReInitDelayCount`/`C_SocReInitCurrentAbsF` 사용처 확인 후 정리.
6. **임시 TODO 주석** — `main.c`·`PWRHoldHandle`에 "검증 후 제거" TODO 다수. 검증 완료 후 제거.
7. **빌드 산출물 형상관리** — `Debug/` 산출물 커밋 정책 확정 권장(`.gitignore` 여부).

## 6. 결론

부팅 1회 OCV 초기화 + 운용 중 전류적산 + PWRHOLD 전원 유지의 3단 구조로, **전원 재투입 시 SOC 점프**라는 기존 문제를 구조적으로 차단했다. 위 검증 계획(A~E) 완료 후 양산 반영을 권고한다.
